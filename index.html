Web server

For this project, I (along with another student), implemented a multithreaded web server in C. The web server uses POSIX threads, and directly accesses the Unix socket system call interface to respond to HTTP GET requests, utilizing and pool of dispatcher threads to process the request and a pool of worker threads to respond to the request with the appropriate file data. It also includes an implementation for a basic cache, as well as a dynamic thread pool system that adjusts the number of worker threads depending on the current server load.

C, Spring 2022

Honors Thesis

For my honors thesis, I tackled the problem of transit vehicle delay estimation in real time - basically, the live prediction given for when a transit vehicle will arrive at a specific stop. I used Graph Neural Networks to try to make network wide predictions, an approach that I found scarce within urban transit networks in existing literature.

While the approach was not as successful as the existing predictions being made by Metro Transit (a hard thing to beat, considering the timeframe of my project), it nonetheless was accurate enough to show value in the approach in general. For more details, read the paper!

Zillow Bot

Automated web service used to retrieve Zillow home valuation data en-masse using the Zillow API. The service allows (private) users to upload loan tapes in a particularly formatted excel spreadsheet, which it then parsed in order to get a list of properties before searching for those properties on the API, retrieving their Zestimate when possible, and putting that data back into the spreadsheet. All of this was done using a web frontend which made asynchronous callbacks to a Node.js web server which did most of the processing.

Much of this source code is private to protect the interests of the company I made the bot for, but the Zillow API accessing code is available, as well as an example of the front-end interface in action.

Video Cut Generator

This project was born out of a desire to help a friend automate the process of cutting and splicing many video clips from many different sources. He specifically needed to cut small chunks from various sources and splice them together in order, but only knew the rough timestamps of where the cuts needed to be, needing to move frame by frame in order to get the exact location.

The project is split into three parts: generate_cuts, which takes in a CSV file containing approximate timestamps and plays the video so that you can select a final cut location; make_cuts, which actually makes the cuts based off of the output of generate_cuts; and splice, which splices all of them together. splice was never implemented, but the other two features are (and arguably are the most unique part of the project anyways).

Websocket File Transfer

Online P2P file transfer system that I made, which uses WebSockets to transfer files from a sending end to a receiving end. This is done by having the receiving end start the site, and the sending end can then scan a QR code to connect to the receiver, or enter a text code into the site. I made this in order to facilitate an easy way to send files from my phone to my computer, without needing to find a cable or spend the time to login to a service like Google Drive on both devices. The server for this is written in Python, and the web interface in HTML/JS with Bootstrap styling.

Unfortunately, the server that hosted this service is currently down, but the source code is available!
